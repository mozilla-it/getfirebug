No console object in workers - test case

See also main script source and worker source.

A worker starts on window.onload, adding 1 to a variable i = 1 in a loop until i is 5000. When i is a multiple of 100, the worker posts a message to the main script, which adds the current value of i to the list below. When i is a multiple of 500, the worker (in addition) tries to call console.log(), but fails miserably. A try... catch... block notices this, and posts a failure message, which is also added to the list below.
