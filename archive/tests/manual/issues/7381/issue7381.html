Issue 7381: Execution stops on error after reload when 'Break On All Errors' is not active

Steps to reproduce

  1. Open Firebug
  2. Enable the Console and the Script panel
  3. Ensure the Script panel option Break On Exceptions is not checked
  4. Switch to the Console panel
  5. Click the Break On All Errors button
  6. Reload the page
    ⇒ Firebug switches to the Script panel ⇒ The script execution is stopped at the error at line 8 (undefinedFunction();)
  7. Click the Continue button
  8. Switch to the Console panel again
    ⇒ The Break On All Errors button is not pressed anymore.
  9. Reload the page again

Observed result

  • Firebug switches to the Script panel again and the script execution stops again at the error

Expected result

  • Because the Break On All Errors button is not pressed anymore, i.e. breaking on exceptions disabled, the script execution should not stop a second time.
