FireBug Console Scrollbar Bug

Dojo's soria.css will cause a ton of CSS warnings (assuming you have "Show CSS Errors" enabled) followed by the "Doing addOnLoad..." message. Upon a page (re)load, FireBug's console history scrollbar should stick to the bottom showing "Doing addOnLoad..." to start, but it stays at the top instead.

Note: This bug did not exist in FireBug 1.3, but it does exist in many 1.4 versions including the latest 1.4.0b7.
