Enabling Firebug

Firebug tries to stay out of your way until you need it; it goes away if you suspend it. Firebug can be optimized for HTML authors, CSS designers, Javascript developers, and Site Engineers. You can focus on a site or work across sites using a specific Firebug feature.

One Click to Start Firebug

After you install Firebug you will see a gray 'Firebug' in the Firefox status bar (along the bottom of the window). Click on the Bug to start. Firebug will open in the bottom of your web page.

Instant HTML, CSS, and DOM Views

As soon as you open Firebug you can examine the Web page markup, styling, and object model. Three tabs -- HTML, CSS, and DOM -- activate as soon as you open Firebug. If you are a HTML author or CSS designer, this may be all of the Firebug you need. As long as you don't enable other panels, the overhead caused by using the Firebug should be small: by default Firebug keeps all the expensive features off.

Digging Deeper into a Web site

To work on Javascript code or to study the network action on a site, you need to enable one or more Firebug panels. If you select the Console, Script, or Net panels, you will see a control for enabling one or more of these panels for the site you are viewing. Select the panels you need, then push the big button to enable them. (Firebug will reload the page These panels use active listeners and Firebug needs to reload the page to fill the panels with data).

Coming up for Air (or email...)

Once you have enabled one of the active-listener panels, Firebug will cause some overhead on every page. The Firebug team is working to reduce this overhead, but for now Firebug has a quick way to suspend the listeners. Look back at Firefox's status bar: you will see that the Firebug icon is now orange, meaning 'on'. To suspend the active panels, right click on the Firebug icon and select Suspend Firebug; to resume select Resume Firebug or simply open Firebug again.

Firebug For Every Site

Once you are fully addicted to Firebug you can take the next step on enablement: enable panels for all sites.

On the Console tab you will find a mini-menu. If you select Enabled, then the Console panel will be enabled for all sites you visit, unless you explicitly disable the Console for that site.

Firebug for Almost Every Site

When you have Console, Script or Net panel enabled for every site, you might like disable Firebug for a few sites. (This is similar to Firebug 1.0, which was enabled by default but allowed sites to be disabled).

From the panel tab's mini-menu, select Disable for .... The panel will revert to the disabled state.

Why is Firebug Still On???

Whenever the Firebug status bar icon is orange (not gray), Firebug has one or more active listeners. If you get confused about why Firebug is still on, hover over the icon. Firebug may be activated by a tab you currently do not have on top.

Working with Site Permissions

To tweek the enable/disable settings for any panel across all sites, select Sites... from the mini-menu.

Firebug For HTML Authors and CSS Designers

If your primary work in Firebug is with HTML or CSS, leave all the active-listener panels Disabled. Firebug will have minimal overhead when you are not using it. You probably want enable the Console to look for errors. Enable one site at a time or enable the Console (only) for all sites, then selectively disable it for sites like your email

Firebug for Javascript Developers

For Javascript you'll want the Console and Script panels enabled. You may was well run with these panels fully enabled, then suspend Firebug when you want to read e-mail or other wise goof off. Just remember to set the panels for the goof-off sites to Disabled for... so suspend stays suspended.

Firebug for Site Engineers

If your primary job is monitoring load times and looking out for 404s, activate the Net panel and perhaps the Console.
